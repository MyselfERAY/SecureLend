Entegrasyon Servisleri

5 dış servis entegrasyonu — Abstract Service Pattern ile geliştirmede Mock, üretimde Gerçek API kullanılır.

MOCK HAZIR

BankService

KMH hesap yönetimi, transfer ve ödeme emirleri
  • POST applyForKmh(userId, dto)
  • POST acceptOffer(appId, userId)
  • POST startKyc(appId, userId)
  • POST verifyId / verifySelfie
  • POST completeVideoCall(appId)
  • POST signAgreements(appId)
  • POST completeOnboarding(appId)
  • POST notifyContractSigned(contractId)
  • POST transfer(from, toIban, amount)
  • GET getBalance(accountId)
  • GET getAccountsByUser(userId)
  • GET getKycStatus(appId)
  • POST cancelApplication(appId)
MOCK HAZIR

SmsService

OTP gönderimi ve bildirim SMS'leri
  • POST sendOtp(phone, code)
  • POST sendNotification(phone, msg)
Üretim hedefi: Netgsm veya İletimerkezi API
MOCK HAZIR

CreditScoringService

Kredi skorlama ve risk değerlendirme
  • POST checkCredit(tckn, income)
  • GET getCreditScore(tckn)
Üretim hedefi: KKB / Findeks API entegrasyonu
MOCK HAZIR

IdentityVerificationService

Kimlik doğrulama ve KPS entegrasyonu
  • POST verifyIdentity(tckn, name, dob)
  • GET getKpsStatus(tckn)
Üretim hedefi: E-Devlet MERNİS / KPS
PLANLI

PropertyVerificationService

TAKBİS üzerinden mülk sahiplik doğrulama — UAVT bazlı
  • GET verifyOwnership(uavtCode, ownerTckn)
  • GET getPropertyDetails(uavtCode)
  • GET checkEncumbrance(uavtCode)
Üretim hedefi: TKGM TAKBİS Web Servisi
Kullanım: "Bu UAVT numaralı mülk, bu TCKN'ye mi ait?" doğrulaması.
Sözleşme oluşturma sırasında ev sahibinin mülk sahipliği otomatik kontrol edilir. Ayrıca ipotek/haciz kontrolü ile risk değerlendirmesi yapılır.

Güvenlik Mimarisi (10 Katman)

Bankacılık standartlarında güvenlik katmanları — dıştan içe doğru savunma derinliği.

1
Helmet HTTP Headers

XSS, Clickjacking, MIME-sniff koruması

2
CORS Origin Control

Yalnızca izinli frontend URL'leri

3
3-Tier Rate Limiting

Short (10/min) • Medium (100/hr) • Long (1000/day)

4
JWT Auth Guard

Access (15dk) + Refresh (7gün) token çifti

5
Roles Guard (RBAC)

TENANT / LANDLORD / ADMIN rol ayrımı

6
DTO Validation Pipe

class-validator + whitelist + forbidNonWhitelisted

7
TCKN Hash + Pepper

Kriptografik hash ile kimlik saklama

8
Log Masking

TCKN, telefon, OTP otomatik maskeleme

9
Audit Logging

Tüm kritik işlemler + IP + User-Agent

10
AllExceptions Filter

Güvenli hata mesajları — stack trace gizleme

Entegrasyon Yol Haritası

Mock'tan Production'a 16 haftalık geçiş planı — her servis için adım adım entegrasyon.

Faz 1 — Hafta 1-4: Teknik Keşif

API Spesifikasyon & Sandbox

  • Banka API dokümantasyonu inceleme ve spesifikasyon anlaşması
  • Sandbox/test ortamı kurulumu (banka tarafı)
  • MockBankService → Gerçek API mapping tablosu
  • Test veri setleri ve senaryo tanımlama
  • Güvenlik gereksinimleri (mutual TLS, API key, OAuth2) belirleme
Faz 2 — Hafta 5-8: Entegrasyon

Servis Bağlantıları

  • BankService: KMH başvuru, değerlendirme, hesap açılış API'leri
  • SmsService: Netgsm/İletimerkezi SMS gateway bağlantısı
  • CreditScoringService: KKB/Findeks API entegrasyonu
  • IdentityVerificationService: E-Devlet KPS bağlantısı
  • PropertyVerificationService: TKGM TAKBİS entegrasyonu (UAVT bazlı mülk sahiplik doğrulama)
  • Hata yönetimi ve retry mekanizmaları
Faz 3 — Hafta 9-12: Test & UAT

Kalite Güvence

  • Uçtan uca test: KMH başvuru → Onay → Onboarding → Sözleşme → Ödeme
  • Penetrasyon testleri (OWASP Top 10)
  • Performans testleri (yük, stres, dayanıklılık)
  • BDDK ve KVKK uyum kontrolü
  • Banka güvenlik denetimi
Faz 4 — Hafta 13-16: Pilot & Go-Live

Canlıya Geçiş

  • Pilot bölge seçimi (İstanbul — Kadıköy/Beşiktaş vb.)
  • Kademeli kullanıcı artışı (%10 → %25 → %50 → %100)
  • Monitoring, alerting ve on-call yapısı kurulumu
  • Full production go-live ve basın duyurusu
